# Codex Agent Guide

## 目标

本项目是一个减肥主题的微信消消乐小游戏，项目名为 `轻盈甜心消除`。

Agent 在执行任务时，应优先理解并维护以下核心目标：

- 保持小游戏玩法稳定，可正常在微信开发者工具中运行
- 尽量做最小范围修改，避免无关重构
- 尊重现有目录职责，不随意打破 `shared/` 与 `src/` 的边界

## 项目概览

根据 `README.md`，当前项目具备以下特征：

- 微信小游戏原生实现，基于 Canvas 和微信小游戏 API
- 已包含角色选择、体重输入、首页、游戏页、结算页
- 已实现无限 / 经典模式、三消、连击、道具、反弹机制、甜心狂切等玩法

## 目录认知

- `game.js` / `game.json`: 小游戏入口与基础配置
- `app.js` / `app.json`: 应用级配置
- `shared/`: 可复用纯逻辑，例如消除、进度、结算、状态机
- `src/`: 小游戏运行时能力，例如渲染、输入、音频
- `assets/`: 本地资源，例如角色图、音频

## 基本工作原则

1. 先阅读相关文件，再修改代码或文档。
2. 优先复用现有实现风格、命名方式与目录结构。
3. 若只需修复局部问题，不主动扩大改动范围。
4. 涉及玩法逻辑时，优先确认是否应放在 `shared/`。
5. 涉及渲染、输入、音频、小游戏生命周期时，优先确认是否应放在 `src/`。
6. 默认使用本地资源路径，避免引入不必要的外链依赖。

## 修改前检查

在开始任何实现前，建议先确认：

- 本次需求影响的是玩法逻辑、运行时表现，还是资源内容
- 是否已有可复用模块
- 是否会影响经典模式、无限模式或特殊机制
- 是否需要同步更新说明文档

## 交付要求

- 提供简洁的变更说明
- 标明是否进行了验证，以及验证方式
- 若存在未覆盖风险或待确认点，需要明确写出
